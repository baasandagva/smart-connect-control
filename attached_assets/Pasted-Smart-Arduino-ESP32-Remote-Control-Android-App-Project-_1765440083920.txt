This project is an Android application designed to remotely control a smart device built using ESP32 and Arduino-based actuator modules.
The main purpose of this app is to allow the user to connect to the ESP32 through Wi-Fi and remotely send commands to control the smart hardware (doors, switches, relays, motors, sensors, etc.).

Works fully offline inside the same local Wi-Fi network

üîå ESP32 Connection Information

Your Android app interacts with ESP32 using the following communication rules:

Connection Type: Wi-Fi (HTTP protocol)
Default ESP32 endpoint:
http://<ESP32_IP_ADDRESS>/<command>


Example commands:

http://192.168.1.55/open
http://192.168.1.55/close
http://192.168.1.55/on
http://192.168.1.55/off


If your ESP32 uses JSON responses, the app expects:

{"status":"ok","device":"opened"}

üîß How the App Sends Commands to ESP32

Example Kotlin/Java HTTP request:

fun sendCommand(command: String) {
    val url = "http://192.168.1.55/$command"
    Thread {
        try {
            val result = URL(url).readText()
            println("ESP32 Response: $result")
        } catch (e: Exception) {
            println("Error: ${e.message}")
        }
    }.start()
}
